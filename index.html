Motivation

  • Make research findings/data FAIR
  • With detailed attribution and metadata
  • Also support original or personal contributions
  • Contributions can be small, a bit like a semantic version of Twitter
  • But also with support for larger datasets
  • Robust via decentralized architecture

Nanopublication

http://nanopub.org

Nanopublication Principles

  • Represent data in small "atomic" packages
  • Attach provenance and metadata to these packages
  • Represent everything as Linked Data (in RDF)
  • Treat each such small package as independent publication ("nanopublication")

Nanopublications for Non-Public Data

  • Benefits are most pronounced for public data
  • But nanopublications can be applied for non-public data too (but less tool support so far)
  • In particular provenance-focus and granularity can help in both cases
  • Can provide uniform format for integration of public and non-public data
